Titel: Transaktionen, ACID-Eigenschaften und Isolation

1. Transaktion
- Eine **Transaktion** ist eine logisch zusammengehörende Folge von Lese- und Schreiboperationen, die als unteilbare Einheit behandelt wird.

2. ACID-Eigenschaften
- **Atomicity (Atomarität)**: Entweder werden alle Operationen einer Transaktion ausgeführt oder keine.
- **Consistency (Konsistenz)**: Eine Transaktion führt die Datenbank von einem konsistenten Zustand in einen anderen.
- **Isolation**: Parallel laufende Transaktionen beeinflussen sich so, als würden sie seriell ausgeführt.
- **Durability (Dauerhaftigkeit)**: Nach Commit bleiben die Änderungen auch nach Systemfehlern bestehen.

3. Nebenläufigkeitsprobleme
- **Dirty Read**: Lesen nicht dauerhaft gespeicherter (uncommitted) Änderungen.
- **Non-Repeatable Read**: Wiederholtes Lesen derselben Zeile liefert unterschiedliche Werte.
- **Phantom Read**: Wiederholtes Ausführen derselben Anfrage liefert verschiedene Mengen an Zeilen.

4. Isolation Levels (SQL-Standard)
- **Read Uncommitted**: Dirty Reads möglich; nur atomare Zeilenoperationen garantiert.
- **Read Committed**: verhindert Dirty Reads; Non-Repeatable und Phantom Reads möglich.
- **Repeatable Read**: verhindert Dirty und Non-Repeatable Reads; Phantom Reads können möglich sein.
- **Serializable**: verhindert alle genannten Phänomene; entspricht serieller Ausführung.

5. Implementierung von Isolation
- **Zwei-Phasen-Sperrprotokoll (2PL)**: Sperren werden in einer Wachstumsphase angefordert und in einer Schrumpfphase freigegeben; garantiert Konflikt-Serialisierbarkeit.
- **MVCC (Multiversion Concurrency Control)**: Leseaktionen sehen konsistente Schnappschüsse; Schreibaktionen erzeugen neue Versionen von Datensätzen.
